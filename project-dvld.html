DVLD

Completed

A desktop system for managing drivers, license applications, and driving tests — built to make an approval chain explicit in code rather than just policy.

RoleSolo developer
StackC#, WinForms, SQL Server
DurationMar — Apr 2026
Sourcegithub ↗

The problem

A licensing office tracks a lot of interdependent records: who a driver is, which tests they've taken, which licenses they hold, and which staff member is allowed to approve what. In a paper system, or a database with no enforced relationships, it's easy for one of those links to silently break — a license issued without a passed test, an approval granted by someone without the role to grant it.

DVLD's job is to make that chain of dependencies impossible to skip, not just discouraged.

How the data is structured

That meant starting from the schema, not the screens. Every license traces back through an application to a driver — there's no path to create one without the other:

The Users table matters as much as the others: every action a staff member takes is checked against their role before it's allowed to touch any of these tables, so the enforcement isn't just structural — it's also about who's allowed to act.

Keeping logic out of the UI

The application follows a Three-Tier structure so that none of those rules end up living inside a button click handler:

Presentation → WinForms screens, input handling only
Business Logic → validation rules, workflow / approval state
Data Access → ADO.NET + stored procedures

Forms never call SQL directly, and the business logic layer never knows or cares how data happens to be stored — it could be SQL Server today and something else later without the rules themselves changing.

What I'd do differently now

Looking back at this after starting NovaERP, the WinForms client is the part I'd change first — it ties the business rules to a desktop runtime they don't need to depend on. Re-exposing this same domain as a Web API would be a natural next pass, and it's part of why NovaERP starts from that assumption instead.
